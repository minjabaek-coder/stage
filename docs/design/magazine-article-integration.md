# 매거진 ↔ 기사 연동 개선 설계

> 상태: **설계(코드 미착수)** · 작성 2026-07-11
> 관련: [[article-magazine-merge]](./article-magazine-merge.md) · 구성형 에디터 `docs/design/v2/editor-canvafy/`

## 0. 배경 / 현재 구조

- **연결 모델**: `MagazinePage.articleId`(nullable, `onDelete: SetNull`, unique 아님)로 **페이지 1장 ↔ 기사** 연결. 데이터상 **N페이지 : 1기사 이미 가능**.
- **연결 UI**: 구성형 페이지 에디터(`page-editor.tsx`)의 `<select>` — **전체 기사 평면 목록**에서 한 페이지에 하나 선택.
- **"실린 곳" 딥링크**: 기사 상세가 `magazinePage.findFirst({ where:{articleId}, orderBy:{pageNumber:'asc'} })` → **첫 페이지**로 이동(`/magazines/{id}?page={n}`).
- **콘텐츠 이식**: 편집자가 **빈 캔버스에 기사 내용을 수동으로** 블록화(제목/본문/이미지). 공수 큼.
- **프리셋**: `magazine-presets.ts` 5종(표지형/풀사진+캡션/제목+본문/2단/사진+본문).
- **콘텐츠 모델**: 구성형 `PageLayout { blocks, pageBg }`, 블록 좌표 % (2:3 캔버스 440×660 기준), 텍스트 `fontSizePx`(기준폭 440). 기사 HTML은 Tiptap(캡션=`figure`, 갤러리=`div.article-gallery[data-gallery]`).

## 1. 문제 정의

1. **기사 찾기(스케일)** — 전체 기사 평면 드롭다운. 기사가 늘면 못 찾음. *단순 검색 추가로는 근본 해결 안 됨.*
2. **다중 페이지 연동** — 기사가 여러 페이지에 걸쳐도 UI는 페이지별 개별 연결이고, "실린 곳"은 항상 첫 페이지로만 안내(범위 인지 없음).
3. **수동 이식 공수** — 사람이 기사를 처음부터 옮김.

**근본 재정의**: 세 증상의 뿌리는 *매거진 에디터가 기사를 외부 평면 풀로 취급 + 배치 100% 수동*. → **"템플릿 보조 조립"** 으로 묶어 공수를 구조적으로 줄인다.

## 2. 결정사항 (2026-07-11 확정)

| 항목 | 결정 | 비고 |
|---|---|---|
| 주 경로(문제1) | **지금은 매거진 선택기 개선(pull)** | **호 단위 큐레이션**은 적절하나 후속으로. push(기사→매거진)도 후속 후보 |
| 자동 초안(문제3) | **규칙 엔진만** | LLM 미사용(무비용·결정적·예측가능) |
| 다중 페이지(문제2) | **연속 범위(P10–13)** | 비연속 집합 미지원 |
| 착수 | **본 설계 문서 먼저** | 코드는 이후 |

## 3. 문제 1 — 기사 선택기 개선 (pull, 지금)

전체 평면 `<select>`를 **맥락형 선택기**로 교체. 규모가 커져도 실용적이게.

### 3.1 선택기 요구사항
- **타입어헤드 검색**: 제목·저자 부분일치.
- **필터**: 장르, 상태(기본 `published`), **"이 매거진에 미배치"** 토글.
- **메타 표시(행)**: 썸네일·제목·(장르·저자·발행일)·상태 배지 + **"이미 STAGE N호 P10–13에 실림"** 뱃지.
- **정렬**: 미배치 우선 → 최근 발행 우선.
- **현재 페이지 연동 상태** 표시 + 해제 버튼.

### 3.2 데이터
- 이 매거진의 페이지들에서 이미 쓰인 `articleId` 집합을 계산 → "미배치" 필터·"실린 곳" 뱃지에 사용.
- 선택기용 기사 목록은 경량 필드만(썸네일/제목/장르/저자/발행일/status). 본문 제외.

### 3.3 후속(기록만): 호 단위 큐레이션
- "이번 호 기사 세트"를 먼저 큐레이션(가벼운 매거진↔기사 연결) → 페이지 선택기는 **호 세트(소수)** 만 노출. 목록이 애초에 짧아져 스케일 문제 소멸.
- 도입 시 데이터: `MagazineArticle`(magazineId, articleId, sortOrder) 경량 조인 재도입 검토([[article-magazine-merge]]에서 폐기했던 개념의 '큐레이션 전용' 축소판). **지금은 미도입.**

## 4. 문제 2 — 연속 페이지 범위 연동

기사는 **연속된 페이지 범위**(예: P10–13)를 차지한다고 본다.

### 4.1 모델
- 저장은 현행 유지: 범위 내 **각 페이지에 동일 `articleId`** 부여(별도 테이블 불필요).
- "배치(placement)" = 같은 매거진에서 같은 `articleId`를 가진 **연속 페이지 런(run)**.

### 4.2 UI
- 페이지 에디터에 **범위 지정**: "이 기사를 P__–P__에 연동"(시작~끝). 지정 시 범위 전 페이지 `articleId` 설정, 범위 밖은 해제.
- 자동 초안(문제3)으로 만들면 범위가 **자동 부여**됨(가장 자연스러운 경로).
- **가드**: 같은 기사의 페이지가 비연속이면 경고(연속 범위 가정 위반).

### 4.3 "실린 곳" 표시/딥링크
- 기사 상세: `articleId`의 **min/max pageNumber** 계산 → "**STAGE N호 P10–13에서 보기 →**".
- 이동 목적지 = **범위 시작 페이지**(기사 시작점 = 현행과 동일, 올바름).
- (옵션) "대표 페이지" 별도 지정 — **기본=시작 페이지**로 두고 후속 검토.

## 5. 문제 3 — 규칙 기반 자동 초안 ("콘텐츠 → 레이아웃 컴파일러")

**목표**: 빈 캔버스 수동 이식 → *자동 초안 80% + 사람이 다듬기*. LLM 없이 규칙으로.

### 5.1 파이프라인
```
기사(HTML) → ① 흐름 분해 → ② 분량 기반 페이지 분할 → ③ 프리셋 매핑
          → 초안 MagazinePage[](자동 연동·연속 범위) → 편집자 다듬기
```

### 5.2 ① 흐름 분해 (content stream)
기사 HTML을 순서 있는 의미 노드로 파싱:
- `h2`/`h3` → 섹션 제목
- `p` → 문단(본문)
- `figure(.article-figure)` img+figcaption → 이미지+캡션
- `div.article-gallery[data-gallery]` → 갤러리(N장)
- `blockquote` → 인용(풀쿼트 후보)
- 메타: `title`/`subtitle`/`author`(바이라인)

*이미 우리가 만든 캡션·갤러리 구조라 파싱 재사용 가능.*

### 5.3 ② 분량 기반 페이지 분할 (관건, 휴리스틱)
구성형은 고정 2:3 캔버스 + `fontSizePx`라 **텍스트 블록 용량을 추정 가능**.

- 블록 px: `w_px = w%/100·440`, `h_px = h%/100·660`.
- 줄당 글자수 ≈ `w_px / (fontSizePx · K)` (한글 K≈1.0, 라틴 더 작음).
- 줄수 ≈ `h_px / (fontSizePx · lineHeight)`.
- 블록 용량(chars) ≈ `줄당글자수 · 줄수`.
- 본문을 **문단 경계 우선**으로 이 용량에 맞춰 페이지 단위로 분할(문단 중간 분할은 지양, 넘치면 다음 페이지로).

> 1단계는 이 휴리스틱으로 "대략" 분할(초안이라 완벽 불필요, 사람이 다듬음).
> 2단계(후속): 실제 측정 기반 정밀화.

### 5.4 ③ 프리셋 레시피 (규칙)
콘텐츠 흐름을 페이지 템플릿에 매핑:

- **첫 페이지** = 표지/리드: 제목 + 부제 + 바이라인 + (히어로 이미지=기사 썸네일 또는 첫 이미지) + 리드 문단.
- **본문 페이지** = 제목+본문 / 2단 본문: 섹션 제목(있으면) + 용량만큼 문단.
- **이미지 페이지** = 풀사진+캡션 / 사진+본문: 기사 이미지 순서대로, 캡션 유지. 텍스트와 함께 둘지 단독일지는 밀도 규칙으로.
- **갤러리** = 그리드 프리셋(신규 후보): 한 페이지에 N장.

**유동적 페이지 수** = f(총 텍스트 용량, 이미지·갤러리 수, 밀도 설정). 고정 프리셋이 아니라 **콘텐츠가 페이지 수를 결정**.

### 5.5 출력·연동
- 실제 `MagazinePage` 행 생성 + **범위 전 페이지에 `articleId` 자동 설정**(문제 2 자동 해결).
- 생성 위치 = 편집자가 지정한 삽입 지점(현재 페이지 뒤 등).
- 블록은 우리 구성형 모델 그대로라 **뷰어와 동일 렌더**.

### 5.6 재생성 정책
- **1회 생성 후 수동 편집**이 기본. 생성 결과에 손대면 그게 정본.
- **"초안 재생성" 버튼**(경고: 수동 편집 소실) 제공. 자동생성-미편집 여부 플래그로 안전장치 검토(옵션).

## 6. 통합 워크플로 (비전)

> **"기사 → 매거진 스프레드 초안" 액션**이 세 문제를 하나로 수렴:
> - 초안을 **기사 기준**으로 생성 → (지금은 선택기로 고르지만) 배치 시작점이 명확
> - **연속 범위**로 생성 → 다중 페이지 연동 자동
> - **프리셋 초안** → 수동 이식 최소화
>
> 검색/선택기는 "이미 만든 배치를 다시 손보거나 개별 연동"하는 **보조 경로**.

## 7. 데이터 모델 영향

- 문제 1·2: **스키마 변경 없음**(기존 `articleId`·인덱스로 충분). 선택기·표시·범위 UI는 앱 레벨.
- 문제 3: **스키마 변경 없음**(생성물은 기존 `MagazinePage.layout`). 갤러리 그리드 프리셋 추가는 상수만.
- (후속) 호 큐레이션 도입 시에만 `MagazineArticle` 경량 조인 검토.

## 8. 단계(Phase) / 오픈 이슈

**Phase 안(제안, 규칙 엔진 기준)**
- **P1 선택기 개선**: 타입어헤드+필터+메타+미배치 뱃지(스키마 무변경). *가장 즉효.*
- **P2 연속 범위 연동**: 범위 지정 UI + "실린 곳" 범위 표시/시작 이동 + 비연속 경고.
- **P3 자동 초안 v1**: 흐름 분해 + 휴리스틱 분할 + 프리셋 레시피 + 연동 생성 + 재생성 버튼.
- **P4(후속)**: 분할 정밀화(측정), 갤러리 그리드 프리셋, 호 큐레이션, push(기사→매거진) 진입점.

**P3 세부 결정 (2026-07-11 확정)**
1. **휴리스틱 바이어스 = 보수적(여백 여유)** — 페이지를 ~85%만 채우고 넘김(K≈1.05). 넘침(글 잘림)보다 여백을 우선 → 사람은 여백만 다듬음. 값은 실측 후 튜닝 가능.
2. **이미지 배치 = 하이브리드** — 첫 이미지(히어로)·갤러리는 **단독 페이지**, 그 외 인라인 이미지는 **본문과 같은 페이지**(사진+본문). 규칙: 순서상 첫 이미지=히어로(단독), `div.article-gallery`=단독, 나머지=혼합.
3. **대표 페이지 오버라이드 = 불필요** — "실린 곳"은 항상 **범위 시작 페이지**로. (필요 시 후속 추가)
4. **재생성 안전장치 = 경고 다이얼로그만(v1)** — "수동 편집이 사라집니다. 계속?" 확인. 편집추적 플래그는 후속.

**남은 오픈 이슈(구현 중/후속)**
- 휴리스틱 상수(K·채움률)의 실측 튜닝.
- 갤러리 그리드 전용 프리셋 추가 시점(현 프리셋으로 시작 가능).
