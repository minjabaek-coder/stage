## 1. 유틸리티 바 (Utility Bar) — ❌ 제거됨 (2026-06-23 결정)

> **유틸리티 바는 쓰지 않는다.** 날짜·날씨, 패밀리사이트 링크, 상단 인증 링크를 모두 제거.
> 인증(로그인/회원가입)은 **헤더(§3)**에, 패밀리 링크는 **푸터(§5)**에 둔다.
> 아래 원래 사양은 히스토리로만 보존(구현하지 않음).

페이지 최상단의 얇은 잉크 바. 날짜·날씨(좌) + 패밀리 사이트 + 인증 링크(우).

**해부**
```
[ 서울 ☀ 24° · 2027.06.19 목요일 ]            [ ●StageOS ↗ | ●Kairossebook ↗ ‖ 로그인 | 회원가입 ]
        좌(날씨/날짜)                                  우(패밀리)            (우, 인증)
```

| 영역 | 사양 |
|------|------|
| 바 | `bg #1c1b1b`, height `30px`, 내부 1380px 정렬, 좌우 `space-between` |
| 좌측 날씨/날짜 | DM Mono `10px`, `색 rgba(255,255,255,.35)`, `letter-spacing 1px` |
| 패밀리 링크 | DM Mono `9px`, `.35` → hover **골드 `#c4a35a`**. 앞에 브랜드 컬러 점(StageOS 보라 `#6366f1`, Kairossebook 시안 `#0891b2`), 외부링크 `↗` |
| 구분선 | `1px × 14px`, `rgba(255,255,255,.15)` |
| 인증 링크 | DM Mono `9px`, `.4` → hover 골드. "로그인 \| 회원가입" |

**규칙·결정 포인트**
- 유틸바는 **B 서브브랜드 점(보라/시안)** 과 **A 인증 링크**가 한 줄에 공존하는 유일한 곳 → 배경이 잉크 다크라 둘 다 흰색 계열로 통일되어 충돌 없음.
- ❌ **날짜/날씨(좌측) 미채택** (2026-06-23 결정): 유틸바 좌측 날짜·날씨는 **쓰지 않는다.** 좌측 영역은 비우거나 추후 다른 용도(공지/브랜드 태그라인 등)로 재검토. → 구현 시 날짜 표기 제거.
- ⚠️ **인증 상태 연동**: HTML은 비로그인 상태("로그인/회원가입")만. 로그인 시 이 자리를 회원 메뉴(등급 배지)로 바꿀지, 헤더 우측만 쓸지 → 회원 UX 단계에서 정함. 현재 제품은 헤더에 회원 메뉴 보유.
- 모바일: 좁아지면 좌측 날씨/날짜를 숨기고 우측만 유지하거나 바 전체를 접는 안 검토(후속).

## 2. StageOS 배너 (Promo Banner) 🔰

헤더 위, 닫기 가능한 **B 서브브랜드** 프로모. StageOS(제품) 얼리 액세스 유도.

**해부**
```
[✦] Coming Soon · StageOS
    문화예술 기관을 위한 AI 운영 플랫폼 — 모바일 브로셔 · 음성 해설 · …     [얼리 액세스 신청 →] [✕]
```

| 영역 | 사양 |
|------|------|
| 배너 | `linear-gradient(135deg,#0a0f1a,#111827,#0d1520)`, 하단 보더 `rgba(99,102,241,.2)`, `::before` radial 보라 글로우 |
| 아이콘 | `30×30`, `linear-gradient(135deg,#6366f1,#8b5cf6)`, radius `7px`, `✦` |
| 라벨 | DM Mono `8px`, `tracking 3px`, UPPER, `색 rgba(165,180,252,.6)` — "Coming Soon · StageOS" |
| 타이틀 | Noto Serif KR `clamp(12,2vw,15)` 700, `#fff` |
| CTA | `linear-gradient(135deg,#6366f1,#8b5cf6)`, `#fff`, radius `6px`, `7px 18px`, hover `opacity .88` |
| 닫기 | `✕`, `rgba(255,255,255,.3)` → hover `.6` |

**규칙·결정 포인트**
- ✅ **관리자 설정형 배너** (2026-06-23 결정): 현재는 StageOS 고정이지만, **어드민에서 ⑴배너 콘텐츠(라벨·제목·CTA·링크) 편집 + ⑵노출 여부 토글**이 가능해야 한다. → 구현 단계: 배너 콘텐츠를 DB(예: `SiteBanner`/`Announcement` 모델 또는 설정 레코드)로 관리 + 어드민 UI. 노출 OFF면 렌더 안 함. (Phase 4 어드민 작업에 포함)
- 닫기 시 세션 동안 숨김(현재 `sessionStorage`) → **재노출 정책**(영구/세션/N일)은 관리자 설정과 함께 확정(후속).
- 라운드·글로우·그라디언트는 **B 영역이라 허용**. 에디토리얼 본문엔 이 결을 끌어오지 않는다.
- 모바일: 타이틀 말줄임 또는 축약 카피, CTA는 아이콘/짧은 라벨로.

## 3. 헤더 (Header) ⚠️

브랜드 로고 + 주 내비(GNB) + 검색/인증. sticky.

**해부**
```
STAGE   홈  매거진  공연·전시·교육  🎟티켓  🎼AI 마에스트로  STAGE 소개       [🔍] [로그인] [무료 가입]
로고             GNB (active=골드 밑줄)                                        우측 액션
```

| 영역 | 사양 |
|------|------|
| 바 | `bg rgba(252,249,248,.95)` + `backdrop-blur(14px)`, 하단 헤어라인 `rgba(28,27,27,.07)`, `shadow 0 2px 20px rgba(0,0,0,.05)`, **sticky top:0 z:100**, height `58px` |
| 로고 | Noto Serif KR `clamp(20,3vw,26)` 900, `letter-spacing -1.5px`, `#1c1b1b` |
| GNB 항목 | Noto Sans KR `clamp(11,1.2vw,13)` 600, `색 .5`, 하단 `2px` 투명 보더. **hover/active → 잉크 + 골드 `#c4a35a` 밑줄** |
| GNB 특수색 | `🎟 티켓` = 테라코타 `#b5431a` 700(active 밑줄 테라코타) · `🎼 AI 마에스트로` = 틸 `#1b6b6e` 700(active 밑줄 틸) |
| ~~검색 버튼~~ | ❌ **제거됨**(2026-06-23): 통합검색(🔍 오버레이) 미채택. 매거진 목록의 호수·제목 검색은 그대로 유지. 통합검색은 필요 시 후속 재검토 |
| 로그인 | 아웃라인: `border rgba(28,27,27,.2)`, `11px` 700 → hover **잉크 채움+흰글자** |
| 무료 가입 | 솔리드 잉크 `#1c1b1b` `#fff`, `11px` 700 → hover **골드딥 `#6f5c24`** |

**GNB 구조 (IA) — 2026-06-23 확정**
`홈` · `매거진` · `기사` · `티켓` · `소개` · `문의`
- 라우트: `/` · `/magazines` · `/articles` · `/tickets` · `/about` · `/contact`
- **티켓** = 테라코타 특수색(active 밑줄 테라코타), **이모지 없음**. 유일한 "행동(예매)" 메뉴라 색으로만 구분.
- 제외: **공연·전시·교육**(추후 페이지 정리 시 결정), **AI 마에스트로**(도슨트 FAB §7 + 홈 인라인 + 푸터로 접근), **블로그**(푸터). 라벨은 전부 한글.

**규칙·결정 포인트**
- ⚠️ **마이그레이션**: 현재 코드는 공용 그레이 `SiteHeader`(목록/상세)와 홈 인라인 에디토리얼 헤더가 **이원화**되어 있다. v2는 **이 에디토리얼 헤더로 단일화**(모든 공개 페이지 공용 컴포넌트). 버튼/CTA 라벨도 "무료 가입" 등으로 정렬.
- ⚠️ GNB 메뉴 구성이 현 제품(매거진/블로그 2축 중심)과 다름 — "공연·전시·교육", "티켓", "AI 마에스트로"를 주 메뉴로 승격. **각 메뉴의 라우트 매핑**은 page별 명세/구현 단계에서 확정.
- ✅ **모바일 GNB**: 모바일(<md)은 헤더 GNB를 숨김. **인증(로그인/회원가입)은 햄버거 옆에 분리 노출**, 햄버거(☰)는 **내비게이션 메뉴만** 연다(2026-06-23 결정). 헤더 = 로고 + 인증 + 햄버거. 최종 모바일 내비(하단 탭바 §6)는 후속 청크에서 햄버거를 대체.
- ❌ **헤더 통합검색 제거**(2026-06-23): 🔍 버튼·오버레이 미채택. 매거진 목록 자체 검색만 유지.
- 접근성: 아이콘 버튼 `aria-label`, GNB active `aria-current`, 키보드 포커스 링.

## 4. 장르 서브내비 (Genre Sub-Nav) 🔰

헤더 바로 아래, 가로 스크롤 가능한 장르 탭. sticky.

**해부**
```
NEW  클래식  오페라  무용  연극  뮤지컬  국악  전시  교육  🎼 AI추천
(active=골드 밑줄)                                          (AI탭=틸)
```

| 영역 | 사양 |
|------|------|
| 바 | `bg #f8f5f0`, 하단 헤어라인 `rgba(28,27,27,.08)`, **sticky top:58px z:99**, height `38px`, 내부 `overflow-x:auto`(스크롤바 숨김) |
| 탭 | Noto Sans KR `11px` 600, `색 .45` → hover 잉크, `2px` 투명 보더. **active → 잉크 + 골드 밑줄** |
| AI추천 탭 | 틸 `#1b6b6e`, active 밑줄 틸 |

**탭 구성:** `NEW`(최신) · `클래식` · `오페라` · `무용` · `연극` · `뮤지컬` · `국악` · `전시` · `교육` · `🎼 AI추천`

**규칙·결정 포인트**
- 이 장르 축은 [ux-home.md](./ux-home.md)의 **2축 택소노미(카테고리×장르)** 중 "장르" 축이다. 좌측 내비의 장르 목록과 **값이 일치**해야 한다(클래식/오페라/무용/연극/뮤지컬/국악).
- 🔰 **노출 범위 미정**: 장르내비가 홈 전용인지, 매거진/기사 목록에서도 sticky로 쓰이는지 화면별로 차차 확정. (일단 공통 셸 후보로 여기 둠)
- `NEW`/`🎼 AI추천`은 장르가 아닌 **특수 필터**(최신·AI 큐레이션) → 양 끝에 배치.
- 모바일: 가로 스크롤 유지(터치 스와이프). 좌측 내비가 사라지는 760px↓에서 이 바가 장르 탐색의 주 진입점이 될 수 있음(§구멍1 후속 검토 대상).

**규칙·결정 포인트**
- ✅/⚠️ 현 제품 푸터는 이미 4영역으로 개편됨(Wave 1) → **링크 구성·외부 패밀리 노출·발행정보 문구**를 위 IA로 정렬하면 됨(전면 재작성 아님).

## 6. 모바일 내비게이션 — 하단 탭바 + 메뉴 드로어 🔰 (2026-06-23 결정)

데스크탑 헤더 GNB는 `≤760px`에서 숨기고, **하단 고정 탭바**가 주 내비를 맡는다. 상단 장르 서브내비(가로 스크롤)는 그대로 유지(빠른 장르 탐색).

**해부 (모바일 전용, ≤760px)**
```
상단: [장르 서브내비 — 가로 스크롤 유지]
 …본문…
하단: ┌──────────────────────────────────────┐  fixed bottom, z:100
      │  홈   매거진   공연·전시   🎼AI   메뉴  │
      └──────────────────────────────────────┘
                                        └ 탭 → 드로어(바텀시트)
```

| 영역 | 사양 |
|------|------|
| 탭바 | `fixed bottom`, 전폭, `bg paper/95 + blur`, 상단 헤어라인, z `100`. 5탭 균등 |
| 탭 | 아이콘 + 라벨(`10~11px`). active = 골드(🎼AI 탭만 active 시 틸). `aria-current` |
| 5탭 | 홈 · 매거진 · 공연·전시 · **🎼 AI 마에스트로** · 메뉴 |
| "메뉴" 드로어 | 바텀시트(또는 풀 드로어): **카테고리 · 아카이브 · 티켓 · STAGE 소개 · 인증(로그인/가입)**. 데스크탑 좌측 내비·유틸바 인증을 여기로 흡수 |

**규칙·결정 포인트**
- Q1(장르바+드로어) + Q2(하단 탭바) **통합 결과**: 햄버거 아이콘 대신 **하단 탭바의 "메뉴"가 드로어를 연다.** 장르는 상단 장르바, 카테고리·아카이브는 메뉴 드로어.
- 🎼AI 탭이 모바일에서 **도슨트 FAB을 대체**(§7) → 모바일엔 플로팅 FAB 없음(자리 충돌 방지).
- 탭바는 콘텐츠 하단을 가리므로 본문 `padding-bottom`으로 보정. 안전영역(iOS `safe-area-inset-bottom`) 패딩.
- 5탭 한도 내 우선순위: 티켓·소개는 드로어로. 필요 시 탭 구성 조정 가능.

## 7. 도슨트 FAB (마에스트로) 🔰 (2026-06-23 결정)

AI 마에스트로 상시 진입점. **전 공개 페이지 유지**(데스크탑은 플로팅, 모바일은 §6 AI 탭).

| 항목 | 사양 |
|------|------|
| 노출 | **데스크탑(>760)**: 우하단 플로팅 FAB. **모바일(≤760)**: 숨김 → 하단 탭바 🎼AI 탭이 대체 |
| 색/아이콘 | **틸 `#1b6b6e`**(AI 강조색, GNB AI와 일치) + 🎼, 흰색. **보라/펄스(B 서브브랜드) 사용 금지** — AI는 서비스로 읽혀야 함(광고 아님) |
| 모양/위치 | 원형 ~`56px`, `shadow-lg`, 우하단(`bottom`/`right` 여백), 푸터 위로 띄움, z `≥100` |
| 인식 ↑·방해 ↓ | 평상시 라벨 알약("🎼 마에스트로에게 물어보세요") → **스크롤 시 아이콘만 축소**, hover 시 펼침. 첫 방문 1회 가벼운 등장. **스크롤 다운 숨김 / 업 표시.** `prefers-reduced-motion` 존중 |
| 동작 | 클릭 → 도슨트 챗(패널/페이지). 기존 도슨트 챗 컴포넌트 재사용 |

> ✅ "전 페이지 유지"를 데스크탑 FAB + 모바일 AI 탭의 **이중 어포던스**로 충족. 항상 같은 마에스트로로 연결.

## 8. 맨 위로 스크롤 버튼 (Scroll-to-top) 🔰 (2026-06-23, 티켓베이 참고)

긴 3컬럼 페이지용. 일정 이상 스크롤 시 노출.

| 항목 | 사양 |
|------|------|
| 노출 | 스크롤 다운 임계(예: 600px) 초과 시 페이드 인, 상단 근처면 숨김 |
| 위치 | 우하단. **도슨트 FAB과 충돌 회피**: FAB 위(데스크탑) 또는 모바일은 탭바 위. 둘이 겹치지 않게 세로 스택 |
| 모양 | 작은 원형(~40px), 잉크/헤어라인 톤(에디토리얼), `↑`. FAB(틸)보다 시각 비중 낮게 |
| 동작 | 클릭 → 부드러운 상단 스크롤(`prefers-reduced-motion` 시 즉시) |

> 우하단 스택 우선순위(위→아래): 맨위로(작게·약하게) → 도슨트 FAB(틸·강조). 마에스트로가 더 눈에 띄도록.
