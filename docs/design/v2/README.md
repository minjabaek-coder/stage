## 상태 표기 규칙 (문서 전반 공통)

각 명세 항목은 **목표(to-be) 디자인**을 기술하되, 현재 코드와의 간극을 아래 기호로 병기한다.

- ✅ **정본 일치(canonical)** — 현재 코드가 v2 기준에 부합. 그대로 따를 것.
- ⚠️ **마이그레이션 대상** — 현재 코드가 구 언어(v1/그레이/shadcn) 또는 다른 구조로 되어 있어 v2로 전환 필요.
- 🔰 **신규(gap)** — 아직 구현되지 않음(HTML이 새로 제안). 신규 작업 시 본 명세를 따를 것.

> 코드 변경은 이 명세 작성과 분리되어 있다. 본 문서는 "무엇이 정본인가"를 정의하고,
> 실제 리팩터링은 별도 작업으로 진행한다. ⚠️/🔰 표기는 후속 코드 작업의 로드맵 역할을 한다.
