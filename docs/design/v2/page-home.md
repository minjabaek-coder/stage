## A. 좌측 내비 (Left Nav) — 2026-06-23 확정

`position:sticky`, `xl(≥1280)`에서만 노출(그 이하 숨김). 폭 ~200px.
**구성: My STAGE(로그인 시에만) + 아카이브(항상).** 상단 장르바는 홈에서 끔(genre는 /articles 필터용). 카테고리/소분류는 Phase 3.

## D. 최신 기사 그리드 (Article Grid) ⚠️

`repeat(3,1fr)` → `560px↓ 2열`, gap `clamp(10,2vw,16)`.

| 영역 | 사양 |
|------|------|
| 카드 | hover `translateY(-3px)` + 제목 골드딥 변색 |
| 썸네일 | `aspect-ratio 3/2`, **각짐**, 상단 그라디언트 오버레이, 좌상단 분류 배지(`art-badge`) |
| 카테고리 | DM Mono `8px`, `tracking 2px`, UPPER, 분류색(커버=골드/인터뷰=틸/리뷰=테라코타) |
| 제목 | Noto Serif KR `clamp(12,1.4vw,13.5)` 700, **2줄 클램프** |
| 메타 | DM Mono `9px`, `색 #9ca3af` (발행일 `YYYY.MM.DD`) |

**규칙·결정 포인트**
- ⚠️ 현행 블로그/기사 카드(그레이·`rounded-lg`)는 이 각진 에디토리얼 카드로 전환.
- 분류 배지 색은 카테고리 매핑을 따른다. 별점(리뷰 `★★★★★`)은 제목 인라인 허용.
- 데이터: Article + 매거진 기사 병합(현행 "기사 탭 병합" 로직과 정합).

## E. AI 마에스트로 인라인 (AI Inline) 🔰

본문 중간의 다크 카드. **A 에디토리얼의 다크 변주 + B 보라 배지 차용**(회색지대 → A로 분류).

| 영역 | 사양 |
|------|------|
| 카드 | `bg #0e0d0b`, radius `10px`, 보더 `rgba(255,255,255,.05)`, 패딩 `clamp(14,3vw,20)` |
| 헤더 | 🎼 + 타이틀(Noto Serif KR `clamp(14,2vw,17)` 700 `#fff`) + 배지 `Gemini · RAG`(DM Mono `8px`, 보라 `rgba(165,180,252,.6)`, `bg rgba(99,102,241,.15)`, pill) |
| 설명 | `11px`, `색 .35` ("이번 호 기사·공연·StageOS 내용을 학습했습니다…") |
| 추천 질문 | pill 버블(`border .12`, `색 .5`, radius 20px) → hover 골드 보더. 예: "조성진 내한 일정은?" |
| 링크 | "전용 페이지에서 대화하기 →" 골드 `#c4a35a` |

**규칙·결정 포인트**
- 🔰 추천 질문 칩(E5/N4 도슨트 강화)·전용 페이지 연결은 마에스트로 단계에서 상세화.
- ⚠️ 배지 문구 "Gemini" = 현행 엔진. 모델 교체 시 문구 동기화(현재 챗 엔진 Gemini).
- 보라는 **배지에만 한정** — 카드 본체는 잉크 다크 유지(A 영역 규칙 준수).

## F. 이달의 공연·전시·교육 (Culture) 🔰

3컬럼(공연/전시/교육) `repeat(3,1fr)` → `560px↓ 1열`, gap `clamp(10,2vw,14)`.

| 영역 | 사양 |
|------|------|
| 컬럼 타이틀 | `11px` 700, 이모지(🎭 공연 / 🖼 전시 / 🎓 교육) |
| 카드 | 보더 `rgba(28,27,27,.08)`, radius `6px`, hover **골드 보더 + `shadow 0 4px 16px rgba(0,0,0,.06)`** |
| 이미지 | 높이 `56px`, 이모지/이미지 |
| 제목 | Noto Serif KR `11px` 700 |
| 메타 | `9px`, `색 #7a7060` (일자 · 장소) |
| 할인 표기 | **원가 취소선 + 회원가 + %배지** (티켓베이 패턴). 예: ~~50,000원~~ **40,000원** `20%↓`. %배지/회원가=테라코타, 원가 취소선=`#9ca3af`. 무료는 `MEMBER 무료`(틸). → 공통 규칙 [ux-home §5.1](./ux-home.md) |
| CTA | `8px` 700 — `티켓`/`예매`(테라코타) / `신청`(틸) |

**규칙·결정 포인트**
- 🔰 **회원 할인 표기**가 콘텐츠에 직접 노출 → 등급(member)별 할인율은 CultureEvent 데이터 + 회원 UX(N2)와 연동.
- 🔰 **가격 필드 필요(구현 단계)**: 원가 취소선+회원가 표기를 하려면 CultureEvent에 `price`/`memberPrice`(또는 할인율로 파생) 필드 추가 필요. 가격 미입력 행사는 `MEMBER N%↓`만 표기(폴백).
- 공연=티켓, 교육=신청으로 CTA 색/문구 분리(테라코타 vs 틸).
- 상세 페이지 부재가 현행 갭(메모리 참조) → 각 카드 클릭 라우트는 구현 단계에서 확정.
