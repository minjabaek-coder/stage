## 2. 핵심 인사이트

1. **우리 두 종류 = 출판 표준 이분법과 정확히 일치.** 이미지형 1~38호 = EPUB **Fixed-Layout**(Issuu/Zinio PDF/Apple News PDF이슈), 구성형 39호+ = **Reflowable/자유캔버스**(ANF/Canva). → 리뉴얼은 "통합"이 아니라 **공통 셸 + 종류별 분기 렌더**가 정석. 억지 통합은 회귀 리스크만 키움.
2. **벤치마크가 한 점에 수렴.** 캔바·미리캔버스·Express 에디터는 모두 "콘텐츠=페이지 배열, 페이지=고정 캔버스 위 절대좌표 요소" — 우리 현행 `MagazinePage(sortOrder)` + `layout.blocks(x/y/w/h/z)`와 **본질적으로 동형**. 대수술이 아니라 (1)px→상대단위 정리 (2)`layout.version` 도입 (3)멀티페이지 패널 UI 추가가 핵심.
3. **피드백 "콘텐츠 단위 멀티페이지" = Express 멀티페이지 HUD + Canva 그리드뷰와 1:1.** 현행은 페이지마다 라우트 전환 → **단일 에디터 셸 안 페이지 썸네일 패널로 라우트 전환 없이 스위칭**이 1순위.
4. **절대좌표는 본질적으로 반응형이 아니다.** AI Magic Resize 같은 비결정·과금 의존 배제. **결정론 규칙**으로: 데스크톱=고정 캔버스 균일 scale, 모바일=블록 order 기반 1열 폴백 + 블록별 오버라이드(Figma "먼저 자유롭게, 필요한 곳만 제약").
5. **px 좌표/글자 저장 금지가 가장 중요한 데이터 규칙.** 현행 `fontSizePx`(기준폭440)를 상대단위(cqw/baseWidth 비율)로 통일 + `layout.version`으로 무중단 읽기호환.
6. **`layout`이 `Json?` 컬럼인 게 안전한 진입로** — 스키마 마이그레이션 없이 코드(`parsePageLayout`)에서 구조 진화 흡수. db push 금지·공유 DB 제약과 정합.
7. **목차는 EPUB nav 원칙(렌더와 분리)으로.** 현행 `MagazineTocEntry`(pageNumber 느슨 매칭)를 **페이지 FK 기반 공통 locator**로 재설계 → 이미지형/구성형 동일 목차 UI + 자동생성.
8. **확대 제스처는 업계 표준 관용구 그대로**(HIG/Material): 핀치=줌, 더블탭=2x 부분줌→복귀(Zinio), 스와이프=넘김. 단 "제스처는 보완, 보이는 컨트롤이 본체" — 줌/목차/페이지 버튼 항상 가시화.
9. **롱폼 모바일 단일 무한스크롤은 최악**(NN/g: 모바일 이해난도 2배). 섹션/페이지 분할 + 목차 점프 + 진행률로 보강. 핵심 진입점은 상단 40%.
10. **접근성은 한계선**(WCAG 2.2·EAA). 구성형은 320px 단방향 스크롤(1.4.10)·텍스트 200% 무손실(1.4.4)·간격 조정(1.4.12) 만족 → 본문 단일컬럼 리플로우. 이미지형은 페이지별 alt/캡션이 최소 방어선(RAG/SEO/Text폴백 자산으로 재활용).

## 3. 콘텐츠 모델 — 3가지 옵션

### A. 자유캔버스 절대좌표 (Canva/Express형 순수)
모든 종류를 절대좌표 한 모델에 흡수(이미지형=배경이미지 단일블록, 구성형=블록배열). 좌표·글자 모두 상대단위(%).
- **장점:** 캔바식 에디터와 1:1, 현행 blocks와 거의 동형(마이그레이션 저비용), 에디터-뷰어 일관.
- **단점:** 진정한 반응형 아님(모바일 균일 scale로 글자 작아짐), 구성형이 사실상 fixed-layout 되어 **WCAG 리플로우/텍스트확대 위반**, 롱폼 가독성·SEO 약함.
- 하위호환 높음 / 모바일 적합 **낮음~중간**.

### B. 블록기반 반응형 (Notion/Ghost형 세로흐름)
좌표 버리고 블록 트리(type+content+order)만. 페이지는 page-break, 런타임 단일 세로흐름.
- **장점:** 반응형·하위호환·접근성이 **데이터 모델에서 공짜**, 모바일 가독성 최고, RAG/SEO 유리, 구현 부담 최저.
- **단점:** **잡지식 2D 자유배치 표현력 상실**(에디토리얼 정체성 충돌), 피드백의 "캔바식" 요구 직접 충족 못함, 구성형 39호+ 디자인 다운그레이드.
- 하위호환 중간 / 모바일 적합 매우 높음.

### C. 하이브리드 — 종류별 분기 + 옵트인 반응형 (Figma+Zinio/Apple News형) ⭐추천
통합 않고 **공통 셸 위 분기**:
- **(a) 이미지형 `kind=image`**: 픽셀고정 페이지 **그대로 보존**(절대 안 건드림), 뷰어는 폭맞춤 스크롤/스프레드 + 핀치/더블탭 줌 + 선택적 focus crop 가이드줌.
- **(b) 구성형 `kind=composed`**: `layout`을 `{ version, canvas:{baseWidth,aspect}, blocks:[{id,type,order,content, layout:{mode:'flow'|'absolute', x/y/w/h Pct(상대단위), constraints, mobile:{order,hidden,fullWidth}}}] }`로 진화. **데스크톱=자유배치 절대좌표 캔버스, 모바일=order 기반 1열 폴백 + 블록별 오버라이드.** 글자도 상대단위. `layout.version`으로 구버전 읽기호환.
- **장점:** 하위호환 최상(이미지형 무변경, 구성형은 Json 안에서 읽기호환 진화=db push 불필요), 에디터 자유배치(데스크톱)+뷰어 모바일 반응형/접근성 **둘 다 충족**, WCAG 충족, **AI 비용 없음**(결정론 규칙), v2 일관성(종류별 분기하되 목차·줌·제스처 규약 공통화).
- **단점:** 데스크톱 자유배치↔모바일 폴백 이중관리 운영부담(→ 모바일 자동 1열을 **기본값**, 오버라이드는 예외로 최소화), layout version 분기 파싱·Zod 검증 구현비, 에디터를 데스크톱 우선으로 범위 한정 필요.
- 하위호환 **최상** / 모바일 적합 높음.

### 추천: **옵션 C**
근거: ① 하위호환 1순위(이미지형 렌더경로 무변경, 구성형만 `layout` Json 읽기호환 진화, 정규 스키마 변경 거의 없음·`migrate deploy`만) ② 요구 동시 충족(캔바식 자유배치=데스크톱 절대좌표, 뷰어 모바일=order 1열 폴백) — A는 모바일/접근성 희생, B는 표현력(에디토리얼 정체성) 희생 ③ 출판 표준 정합(Apple News+ PDF이슈 vs ANF, Zinio PDF vs Text) ④ AI 비결정·과금 의존 배제.

**처음부터 박을 가드레일 5:**
1. px 좌표/글자 금지 → 전부 상대단위(%/cqw·baseWidth 비율)
2. `layout.version` 도입 + `parsePageLayout` 구버전 읽기호환
3. 모바일은 "자동 1열 폴백 **기본값**", 블록별 오버라이드는 예외만(이중작업 회피)
4. 에디터=데스크톱 우선 / 뷰어만 모바일+데스크톱 전면
5. Zod 블록 타입별 엄격 검증(zod/v4 서버액션 패턴)

**1차 범위:** 단일 에디터 셸 + 페이지 썸네일 패널 + 드래그 재정렬 + 기본 스냅 + 절대좌표 드래그/리사이즈 + 모바일 flow 폴백 + 공통 목차/줌. **제외:** 캔바급 스마트가이드, Magic Resize, 자동 panel-guided 좌표 일괄생성, undo/redo.

## 4. 에디터 방향 (데스크톱 우선)

1. **단일 에디터 셸로 통합** — 페이지별 라우트(`/pages/[pageId]` push) 폐기, 한 화면 안 좌/하단 **페이지 썸네일 패널**로 라우트 전환 없이 스위칭. 편집 중 다른 페이지 항상 보임(Express HUD + Canva).
2. **페이지 패널 두 모드** — (1) 썸네일 레일(빠른 전환) (2) 그리드뷰(대량 재정렬·복제). dnd-kit 드래그(현행 `PageListSortable`/`ComposedPageManager` 자산 재활용), 모바일 ↑↓ 폴백 유지.
3. **페이지 컨텍스트 메뉴 표준** — 빈 페이지 추가 / 복제 / 삭제 / 위·아래 이동 / 제목 설정. 빈페이지 vs 복제 명확 구분. 한국어, admin shadcn 톤.
4. **캔버스 편집 자유도 상향** — 8방향 핸들(Shift=비율유지), 드래그 이동, **스냅 가이드**(요소·캔버스 기준), Position 패널(상대단위 직접입력+비율잠금), z-order, 정렬. moveable+selecto 또는 react-rnd 후보. 키보드 nudge·다중선택 가능하면 포함, undo/redo는 차순위.
5. **텍스트 인라인 WYSIWYG** — raw HTML textarea → Tiptap(블로그 자산 재활용 검토). 글자 상대단위 저장.
6. **이미지 프레임/마스킹** — 39호 focus crop(focusX/Y·zoom·fit)을 프레임(클립) 개념으로 일반화. 자산 업로드 경로 매거진 전용으로 통일 검토.
7. **모바일 오버라이드는 별도 탭/모드** — 데스크톱 자유배치가 정본, 모바일은 "자동 1열 폴백 미리보기 + 블록별 순서/숨김/전체폭 예외". 강제 이중작업 금지.
8. **저장 안정성** — 자동저장(드래프트) + 이탈 시 미저장 경고. P2028 한도로 페이지 단위 부분저장 우선(현 2-pass 패턴 존중).
9. **이미지형은 캔버스 강제 안 함** — 업로드·정렬·파일명·삭제 유지하되 같은 페이지 패널 셸에 흡수("한 콘텐츠=여러 페이지" 멘탈모델만 통일). image→composed 전환은 별도 옵트인 액션.
10. **가드레일** — 데스크톱 우선, 죽은 web enum 정리, layout.version, Zod 검증. 캔바급 스마트가이드/Magic Resize/AI 1차 제외.

## 5. 뷰어 방향

### 데스크톱
1. **공통 셸 + 종류별 분기 렌더**(Apple News+) — 외곽(헤더/진행률/줌/목차/페이지 컨트롤)은 하나, 내부 렌더·목차만 kind 분기. page.tsx 하드코딩 분기·magazineId 미전달 버그·데드코드 정리.
2. **이미지형** — 데스크톱 양면 스프레드 유지 또는 폭캡 중앙컬럼 세로스크롤(호별 선택). 키보드(↑↓/PgUp·Dn)+하단 썸네일 필름스트립.
3. **구성형** — 절대좌표 고정 캔버스를 폭에 균일 scale(현 ComposedPage), 최대폭 캡. 텍스트는 실제 텍스트(검색·복사·접근성).
4. **확대** — 인라인 줌을 1급으로(슬라이더/버튼/휠/더블클릭). 별도 라이트박스 진입의 직관성 약점 해소. 줌 컨트롤 공통.
5. **목차 nav 분리·공통화** — 페이지 FK locator 재설계. 이미지형=썸네일 그리드, 구성형=섹션/블록 목차. 우측 사이드패널+하단 썸네일 스트립.
6. **페이지이동** — 좌우 클릭영역+하단 버튼+키보드+썸네일 점프. Kindle Page Flip식 "위치 보존 미리보기"를 썸네일 패널로(에디터와 동일 컴포넌트).
7. **진행률·이어보기** — 하단 진행률바 + 마지막 위치 복귀(블록ID/페이지 앵커 기반, 픽셀 오프셋만 저장 금지). 딥링크 `?page=` 유지.
8. **리더 설정(최소 세트)** — 보기모드 토글, 다크모드, 풀스크린. 구성형은 추가로 글자크기>줄간격>여백>테마(Readium CSS 변수→Tailwind v4). localStorage 영속.
9. **flip/paged 정리** — 종류당 기본 모드 확정, SSR 깜빡임 제거. 책넘김 애니는 데스크톱 선택 옵션.

### 모바일
1. **제스처 표준 공통 강제**(HIG/Material) — 탭=오버레이 토글, 스크롤=UI 자동숨김(웹툰식), 핀치=줌, 더블탭=2x 부분줌→복귀, 스와이프=넘김. **단 가시 컨트롤도 항상 제공**(숨은 제스처 전용 금지), 최초 1회 힌트.
2. **이미지형** — **단면 페이지 스와이프(한 장씩 좌우 넘김)** 1차 모드. 각 페이지는 설계된 단위이므로 페이지 정체성·의도된 레이아웃을 보존(웹툰형 세로 연속 스크롤은 **미채택** — 잡지 페이지 경계가 뭉개짐, 사용자 결정 2026-06-26). 핀치/더블탭 줌+탭→풀스크린 필수. 현행 react-pageflip의 모바일 역넘김 CSS 3D 우회(취약)는 폐기하고 깔끔한 단면 스와이프 컴포넌트로.
3. **이미지형 가이드줌(선택)** — focus crop을 페이지 내 panel-by-panel 가이드줌으로 확장(ComiXology). 좌표 있으면 영역별, 없으면 페이지 전체. 38개 호 일괄 소급 강제 금지.
4. **구성형** — 균일 scale로 욱여넣지 말고 **블록 order 1열 flow 폴백**(WCAG 1.4.10). 블록별 mobile 오버라이드 존중. 텍스트 확대는 줌 대신 글자크기 조절(리디/Kindle).
5. **목차** — 하단 시트/캐러셀(이미지형=썸네일, 구성형=섹션). 단일 무한스크롤 금지, 섹션 분할+점프+진행률.
6. **성능** — 가상화/지연로딩(현행 전 페이지 동시 렌더 해소), 썸네일 경량 별도. 줌 중 스크롤/넘김 잠그는 상태기계로 제스처 충돌 방지.
7. **위치 복귀·호 간 연속** — 이어보기 + 호 끝→다음 호 배너(카카오식 1탭) 검토.
8. **설정 발견성** — 핵심 토글을 진행 오버레이에 직접 노출(설정 하단에 묻지 말 것).
