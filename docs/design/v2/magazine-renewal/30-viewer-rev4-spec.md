## 1. F1 — 홈 표지 폴백 (구성형)

**문제:** 홈 hero(`src/app/page.tsx`)·아카이브 카드는 `magazine.coverImageUrl`만 렌더. 구성형(39호+)은 `coverImageUrl`이 비어 있어 검은 바탕 'STAGE' 플레이스홀더가 표시됨. 실제 표지는 page 1의 `layout`(구성형 디자인)으로 존재.

**해결:** 공통 **`MagazineCover`** 표현 컴포넌트를 도입(또는 기존 표지 렌더 지점에 분기):
1. `coverImageUrl` 있으면 비트맵 `<img>` (기존 동작 유지).
2. 없고 `contentType === "composed"`이면 **page 1의 `layout`을 `ComposedPage`로 렌더**.
3. 둘 다 없으면 기존 'STAGE' 플레이스홀더.

**데이터:** 홈/아카이브 쿼리(`src/app/page.tsx`, `PastMagazines`)가 구성형의 **첫 페이지 `layout`**을 포함하도록 `include`/`select` 추가 (최신호 + 아카이브 카드). 기존 비트맵 매거진은 page 미포함이어도 됨(coverImageUrl 사용).

**적용 지점:** 홈 hero(최신호 표지), `magazine-card.tsx`, `past-magazines.tsx`/`magazine-grid.tsx`. 비율 `aspect-[3/4]`(현행) 유지, `ComposedPage`는 `object-fit: contain` 느낌으로 캔버스 폭맞춤(2:3 캔버스를 3:4 박스 중앙 배치).

## 2. F2 — 목차 = 하단 가로 필름스트립 (데스크톱·모바일 공통)

> **supersede 20-spec §2.4** "데스크톱 우측 사이드패널 / 모바일 하단 시트".

**구조 = 레이아웃 밴드(오버레이 아님).** 뷰어를 세로 플렉스로 보고:
```
[리더 헤더] / [페이지 영역 flex-1] / [필름스트립 밴드 (목차 ON일 때만)] / [컨트롤 바]
```
- 목차 ON → 필름스트립 밴드가 끼어들며 **페이지 영역이 그만큼 줄어듦**. 페이지는 `max-height:100%`라 **전체가 항상 보임**(가림 없음). 목차 OFF → 밴드 제거, 페이지 복귀.
- **데스크톱·모바일 동일 컴포넌트**(반응형). 기존 `TocPanel`의 `isMobile` 분기(세로 패널/시트) 제거.

**스트립 항목:** 가로 스크롤, 각 항목 = **썸네일(aspect 2/3) + 쪽번호**. **제목 텍스트 생략**(작은 썸네일에서 가독성 저하 → 쪽번호만). 현재 페이지 **골드 테두리 + 링** 하이라이트, 열릴 때 현재 항목 자동 `scrollIntoView({inline:'center'})`.
- 썸네일 렌더: `kind==="composed"` → `ComposedPage(layout)`, 이미지형 → `<img imageUrl>` (기존 로직 재사용).
- 데이터 소스: `tocEntries` 우선이되 항목은 **모든 페이지** 기준으로 구성(목차 = 페이지 네비게이터). `tocEntries`가 있으면 제목 메타로 보유하되 표시는 썸네일+쪽.
- 클릭 → 해당 페이지로 `navigateToPage`, 스트립은 열린 채 유지(연속 탐색).

**높이(권장):** 데스크톱 ≈ 112px, 모바일 ≈ 84px(썸네일 한 줄 + 쪽번호). 반투명 짙은 배경 + 상단 1px 보더.

## 3. 모바일 단순화 (F3·F4·F5)

> **supersede 20-spec §2.5/§2.6** 모바일 컨트롤·넘김 관련.

- **F3 확대 버튼 제거** — 하단 컨트롤에서 ⊕ 확대 삭제. **핀치 줌/더블탭 제스처는 유지**(`usePinchZoom`). 라이트박스(`MagazineZoomLightbox`)는 데스크톱 전용 또는 핀치로 대체.
- **F4 넘김 효과 = 항상 즉시 전환** — 모바일에서 `flipEffect` 강제 false(넘김 애니메이션 없음). **넘김(📖) 버튼 제거**. (역넘김 CSS 3D 오버레이도 불필요 시 정리.)
- **F5 자동 전체화면** — 뷰어 진입(모바일) 시 `requestFullscreen()` 시도. 브라우저가 **사용자 제스처를 요구**하면(iOS Safari 등 `requestFullscreen` 미지원/거부) **조용히 폴백**(에러 무시, 기존 화면 유지). iOS는 fullscreen API 제약이 크므로 폴백 필수.
- **모바일 하단 컨트롤(최종):** `목차 · 다크 · 전체` (3개) + 진행률 스크러버 + `N / M`.

## 4. F6 — 모바일 하단 진행률 = 드래그 스크러버

**상단** 진행률(리더 헤더 골드 언더라인) = **표시 전용** 유지.
**하단** 진행률 바 = **인터랙티브 스크러버**:
- 트랙 위에 **손잡이(knob)** 표시(현재 위치 = `currentPage/total`).
- **드래그**: 포인터 X → 비율 → 목표 페이지 인덱스(`round(ratio*(total-1))`) 미리보기, 놓으면 `navigateToPage`. 드래그 중 `N / M` 라벨 실시간 갱신(즉시 전환이라 페이지도 따라가도 됨).
- **탭**: 트랙 탭 위치 비율로 해당 페이지 점프.
- 접근성: `role="slider"` + `aria-valuemin/max/now`, 키보드 ←→(데스크톱은 기존 페이지 이동과 중복이라 모바일 위주).
- 구현 주의: 드래그는 포인터 캡처, 스와이프 제스처(페이지 넘김)와 **이벤트 충돌 방지**(컨트롤 영역은 페이지 제스처에서 제외).
