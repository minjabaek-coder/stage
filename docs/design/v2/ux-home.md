## 1. 정보구조 (IA) — 2축 택소노미

홈은 콘텐츠를 **2개의 독립 축 + 시간 축**으로 탐색하게 한다.

| 축 | 값 | 노출 위치 |
|----|-----|----------|
| **대분류 = 장르** (분야) | 클래식 · 오페라 · 무용 · 연극 · 뮤지컬 · 국악 · 전시 · 교육 | **장르 서브내비**(상단 sticky) + 기사 카드 라벨 |
| **소분류 = 유형** (성격) | 리뷰 · 공연소식 · 인터뷰 · 프리뷰 · 기획·특집 · 칼럼·에세이 · (직접 입력) | 기사 목록 헤더 우측 **드롭다운** |
| **시간** (호수) | 연/월 → 발행호(Vol.) | 좌측 내비 아카이브(연도 접기) |
| **특수 필터** | `NEW`(최신), `🎼 AI추천`(AI 큐레이션) | 장르 서브내비 양 끝 |

**핵심 규칙**
- 대분류(장르)와 소분류(유형)는 **직교한다** — 한 기사는 "클래식(장르) × 리뷰(유형)"처럼 양쪽을 동시에 가지며, 필터는 **AND 결합**(장르 ∧ 유형).
- **장르 값은 한 벌**이다: 좌측 내비 장르 ≡ 장르 서브내비 탭. 한 곳에서만 추가/변경하고 다른 곳이 따라간다(단일 출처).
- `전시`·`교육`은 공연 장르는 아니지만 **공연·전시·교육 콘텐츠 묶음**과 맞물려 장르 서브내비에 포함된다.
- `NEW`/`AI추천`은 분류가 아니라 **뷰(정렬·큐레이션)** → 양 끝에 분리 배치.

> 🔰 데이터 모델 영향: 현행은 매거진/블로그 2축 중심. 위 2축 택소노미를 담으려면 콘텐츠에 `category`·`genre` 필드(또는 태그 분류)가 필요. **스키마 반영은 구현 단계 결정** — 본 명세는 "이렇게 탐색돼야 한다"만 규정.

> ✅ **Phase 3 기사 분류 사양 (2026-06-23 확정)** — 대분류 × 소분류 2단, AND 필터:
> - **대분류 = 장르**: 상단 장르 서브내비(전체·클래식·오페라·무용·연극·뮤지컬·국악·전시·교육). **현행 유지**(탭 변경 없음).
> - **소분류 = 유형**: 리뷰·공연소식·인터뷰·프리뷰·기획·특집·칼럼·에세이 + **직접 입력**(작성자 자유 작성, 관리자가 통계 보고 정식 항목으로 승격). "기사 / N개의 기사" 라인 **우측 드롭다운**(클릭→펼침, 단일 선택 + "전체").
> - **필터 결합**: 장르 ∧ 유형 (AND).
> - **데이터 모델**: `Article.genre`(대) + `Article.subCategory`(소; 직접입력은 동일 필드 자유값) 신규 필드. 어드민 기사 작성 폼에 두 드롭다운(소분류는 직접입력 허용).
> - **적용 범위**: 단독 기사(Article)만. 매거진 기사는 추후 결정.
> - **구현(드롭다운 UI·데이터 모델·직접입력)은 Phase 3.** 본 단계(Phase 1)는 장르 탭 비주얼까지만 완료.

---

> 🔰 **홈 인라인 광고 위치 = 관리자 설정형(Phase 4)**: 현재는 본문 맨 끝 고정. 추후 관리자가 "어느 블록 뒤에 노출"할지 설정. 상세는 광고 작업 시.

## 2. 홈 콘텐츠 흐름 (Reading Order)

중앙 본문의 수직 순서 = **우선순위**다.

(2026-06-23 확정 순서)

1. **최신호 히어로** — 이번 호로의 강한 진입(매거진 보기 / AI와 읽기)
2. **매거진 아카이브** — 지난 호 한 줄(데스크탑 5) + 전체보기
3. **최신 기사** — 개별 기사 소비 (데이터 시)
4. **이달의 공연·전시·교육** — 행동 유도(티켓/신청), 회원 할인 (데이터 시)
5. **AI 마에스트로 인라인** — 능동 탐색(질문)으로 전환
6. **광고** — 본문 인라인(위치는 관리자 설정형, Phase 4)

> 비어 있는 블록(3·4)은 데이터 생기면 노출. 우측 컬럼 위젯(광고·티켓·StageOS·뉴스레터)은 흐름과 별개로 상시.

## 3. 탐색 인터랙션

### 3.1 좌측 내비
- 카테고리/장르 클릭 → 해당 분류 목록으로(active = 골드 좌측 보더 + 골드딥 텍스트). 단일 선택.
- 아카이브 연도 클릭 → `▶/▼` 토글로 월 목록 펼침/접힘. 기본: 최신 연도만 펼침.

### 3.2 장르 서브내비
- 가로 스크롤(스크롤바 숨김, 터치 스와이프). active = 골드 밑줄(AI추천 = 틸 밑줄).
- 좌측 내비 장르와 **선택 상태 동기화 여부** 결정 필요(🔰): 같은 장르를 양쪽에서 누르면 함께 active 되는지, 독립인지. (권장: 동기화)

### 3.3 "전체보기 →" / "전체 →"
- 각 섹션·위젯 헤더의 더보기 → 해당 목록 페이지. hover 밑줄. 일관 문구 사용("전체보기 →" 본문, "전체 →" 위젯).

### 3.4 카드 hover (✅ 에디토리얼 시그니처)
- 기사/매거진 카드: `translateY(-3px)` + 제목 골드딥 변색.
- 표지: `grayscale → 컬러`.
- 문화 카드: 골드 보더 + 옅은 그림자.
- `prefers-reduced-motion` 시 이동/줌 생략, 색 변화만(접근성 §7).

## 4. 광고 / 서브브랜드 표기 규칙 (네이티브 광고) 🔰

광고와 콘텐츠의 혼동을 막는 것이 **신뢰의 핵심**이다.

| 규칙 | 내용 |
|------|------|
| **명시 라벨** | 모든 광고에 `AD` + `Sponsored` 라벨 필수. 생략 금지. |
| **색 분리** | 광고/StageOS/Family는 **B 보라·다크**로 렌더 → 웜 에디토리얼 콘텐츠와 즉시 구분. |
| **배치** | 기본은 **우측 위젯 컬럼**. 본문 인라인 띠 광고를 넣을 경우에도 위 라벨+색 분리 유지. |
| **StageOS 배너** | 자사 제품 프로모도 동일 원칙(닫기 가능, B 색). |
| **문구** | 광고 문의는 `광고 안내`/`광고 문의`로 일관. |

> 네이티브 광고 유형(StageOS 전면·인라인 띠·모바일 브로셔·티켓 카드)이 늘어나면 각 유형 사양을 본 절에 추가. **콘텐츠를 위장한 광고는 금지.**

## 5. 회원 / 등급 노출 (홈 한정) 🔰

홈에 회원 개념이 드러나는 지점:
- **유틸바/헤더 인증** — 비로그인: "로그인/회원가입" · "무료 가입". 로그인 시 회원 메뉴(등급 배지)로 전환(연동은 회원 UX 단계).
- **회원 할인 표기** — 공연·전시·교육·티켓 카드. 비회원에게도 "회원이면 이득"을 보여주는 **전환 유도**(§5.1).

> 등급 배지 색(guest 회색 / member 청록 / pro 골드)·`TierGate`·마이페이지는 **회원 UX(N2)** 에서 정의. 홈은 "할인 미끼 노출"까지만.

### 5.1 할인 표기 규칙 (티켓베이 참고, 2026-06-23 결정)

이득을 **숫자로 구체화**한다 — 추상적 "N%↓"보다 "얼마→얼마"가 전환에 강하다.

```
~~50,000원~~   40,000원   [20%↓]
 원가(취소선)    회원가      %배지
 #9ca3af       테라코타     테라코타 배경 칩
```
- **원가**: 취소선, `#9ca3af`. **회원가**: 강조(테라코타 `#b5431a`). **%배지**: 테라코타.
- **무료**: `MEMBER 무료`(틸). **교육/일반 할인**: 동일 포맷, 색만 분기(틸).
- 🔰 **가격 데이터 없으면 폴백**: 원가/회원가 미입력 시 `MEMBER N%↓`만 표기. (CultureEvent 가격 필드는 구현 단계 — [page-home F](./page-home.md))
- 색은 **테라코타 유지**(티켓베이의 빨강 커머스 톤 아님). 광고/StageOS(보라 B)와도 구분.

## 6. 반응형 동작 (홈)

[layout §3](./layout.md) 붕괴를 사용자 관점으로:

| 폭 | 좌측 내비 | 우측 위젯 | 본문 |
|----|----------|----------|------|
| 데스크탑(>1100) | 노출 | 노출 | 3열 기사 |
| 노트북(≤1100) | **숨김** | 노출 | 3열 기사 |
| 태블릿(≤760) | 숨김 | **숨김** | 단일 컬럼 |
| 모바일(≤560) | 숨김 | 숨김 | 2열 기사·1열 문화 |

**✅ 모바일에서 사라지는 것들의 대안 (2026-06-23 결정 — [global-chrome §6·§7](./global-chrome.md))**
- **장르** → 상단 장르 서브내비(가로 스크롤) 유지.
- **카테고리·아카이브·티켓·STAGE 소개·인증** → 하단 탭바 **"메뉴" 드로어**(바텀시트).
- 헤더 GNB → **하단 고정 탭바**(홈·매거진·공연·전시·🎼AI·메뉴).
- 도슨트 FAB → 모바일은 탭바 **🎼AI 탭**으로 대체(플로팅 FAB 없음).
- 🔰 잔여: 우측 위젯의 **뉴스레터·StageOS·광고**를 모바일 본문 어디에 인라인 삽입할지는 홈 구현(Phase 2) 시 확정(현재는 숨김).

## 7. 접근성 (홈)

- **아이콘 버튼**: 검색(🔍)·닫기(✕)·CTA 이모지에 `aria-label`. 이모지는 장식이면 `aria-hidden`.
- **내비 상태**: GNB·장르탭·좌측 내비 active에 `aria-current`. 골드 밑줄/보더만으로 상태를 전하지 않고(색맹 대비) 텍스트 굵기·`aria-current` 병행.
- **키보드**: 모든 링크/버튼 포커스 가능 + `focus-visible` 링(골드). 장르 서브내비 가로 스크롤도 키보드 도달 가능하게.
- **대비**: 보조 텍스트의 낮은 투명도(`.28`~`.4`)는 작은 글자에서 대비 부족 위험 → 본문·핵심 메타는 `≥4.5:1` 확보 점검.
- **모션**: `prefers-reduced-motion`에서 hover 이동·줌·StageOS 펄스 정지.
- **콘텐츠/광고 구분**: 광고 영역에 라벨 텍스트(`광고`/`Sponsored`)를 시각뿐 아니라 스크린리더에도 노출.

## 8. 상태 (Empty / Loading / Error)

| 상황 | 처리 |
|------|------|
| 데이터 로딩 | 카드 자리 스켈레톤(웜 톤, 각짐). 레이아웃 시프트 방지(이미지 `aspect-ratio` 고정). |
| 빈 섹션 | 해당 본문 블록 숨김 또는 "준비 중" 안내(과한 빈 박스 금지). |
| 이미지 부재 | `bg-dark1~8` 그라디언트 + 워드마크 폴백([tokens §1.C](./design-tokens.md)). |
| 뉴스레터 제출 | 성공/실패 **토스트**(한국어) + 입력 비활성화 중복 제출 방지. 동의 처리.(G2) |
| 외부 위젯(날씨) | 데이터 없으면 위젯/항목 미노출(레이아웃 깨짐 방지). |
